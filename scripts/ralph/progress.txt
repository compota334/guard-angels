## Codebase Patterns
- Project uses ESM (`"type": "module"` in package.json). All imports need `.js` extensions for TypeScript NodeNext resolution.
- tsconfig: strict, ES2022, NodeNext module/resolution, outDir=dist, rootDir=src.
- Vitest configured with `passWithNoTests: true` in `vitest.config.ts` so zero-test runs exit 0.
- ESLint uses flat config (`eslint.config.js`) with `@eslint/js` + `typescript-eslint`.
- Verification commands: `npx tsc --noEmit`, `npx vitest run`, `npx eslint . --quiet`.
- tsconfig has `"types": ["node"]` — required for `process`, `Buffer`, etc.
- CLI entry: `src/bin/angels.ts` (shebang) → imports `src/cli.ts` (commander program). Build: `npm run build` → `dist/bin/angels.js`.
- ESLint enforces `preserve-caught-error` — always pass `{ cause: err }` when wrapping caught errors in new Error.
- Config loading: `src/config/load.ts` → `loadConfig(cwd)`. Schema: `src/config/schema.ts`. Defaults: `src/config/defaults.ts`.
- Path utilities: `src/paths/layout.ts` (canonical .angels/ paths), `src/paths/resolve.ts` (angel-id ↔ path conversion).
- Angel ID encoding: `/` → `-`, literal `-` → `--`. Root angel: `_root` ↔ `.`. Decoder uses char-by-char greedy parse (`--` = escaped hyphen, `-` = separator). Segments consisting entirely of hyphens are rejected by `pathToAngelId` since they cannot round-trip.
- Angel registry: `src/angels/registry.ts` — `AngelRegistry.fromConfig(config)`. Pure in-memory. Lookup by id/path, list all, get root. Validates uniqueness + root constraints.
- Commands go in `src/commands/<name>.ts`, wired into `src/cli.ts`. Each exports a function taking `cwd: string`.
- Synthetic test fixture at `tests/fixtures/projects/sample/` — has `.angels/_config.yml` with 3 angels (_root, src-auth, src-api), angel.md files for _root (active) and src-auth (draft), no angel.md for src-api.
- Integration tests use `execaNode` from `execa` to run the built CLI. Build beforeAll in the test file. Use `nodeOptions: []` to avoid inheriting vitest's loader.
- Folder heuristics: `src/angels/identify.ts` → `identifyCandidates(projectRoot, opts?)`. Ingest: `src/angels/ingest.ts` → `detectExistingMemory(folderPath, projectRoot)`.
- Backend adapters: `src/backend/adapter.ts` (interface), `claude.ts`, `codex.ts`, `droid.ts`, `generic.ts` (implementations), `factory.ts` (`pickAdapter(config)` → adapter). All use execa with `input`/`reject:false`/`timeout`. Factory uses `parseCommandString` to split angel_cmd.
- Test fixtures: `tests/fixtures/echo-backend.sh` (echoes stdin to stdout), `tests/fixtures/stderr-backend.sh` (writes to stderr, exits non-zero). Both are POSIX bash.
- Angel memory: `src/angels/memory.ts` — `readAngelMd(path)`, `writeAngelMd(path, angelMd)`, `updateMetadata(path, partial)`. Atomic write via tmpfile+rename. Zod-validated frontmatter (status, last_updated, last_updated_by). Exports `AngelFrontmatterSchema` and `AngelMd` type.
- Init command: `src/commands/init.ts` — `initAngels(cwd, { auto?, manual? })`. Creates full `.angels/` directory tree, `_config.yml`, `_newspaper.md`, and `angel.md` for each angel. Supports interactive (readline prompt), `--auto` (accept all heuristic candidates), and `--manual` (user enters paths). Ingestion via backend adapter when AGENTS.md/CLAUDE.md exists; falls back to blank template on backend failure.

- Learnings for future iterations:
  - The `replaceAll` approach for decode (`--` → placeholder → restore) is broken when consecutive hyphens create ambiguity. Always use character-by-character parsing for decoding schemes with overlapping escape/separator sequences.
